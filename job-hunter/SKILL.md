# Job Hunter — 多渠道招聘信息检索

## Overview

从主流招聘平台和公司官方招聘页面并行检索岗位信息，按城市/区域/岗位/薪资/距离多维度筛选，输出带距离和来源标注的结构化对比表格。

## 工作流程

当用户提出找工作要求时，按以下五步执行：

### Step 1: 解析需求

从用户输入中提取以下维度：

| 维度 | 示例 |
|------|------|
| 岗位方向 | Java开发、前端、算法、全栈 |
| 地点 | 城市 + 区 + 具体地址（如杭州滨江区康悦香庄） |
| 经验年限 | 3-5年、应届、不限 |
| 薪资预期 | 15-30k、面议 |
| 公司偏好 | 大厂、外企、不加班、指定公司名 |
| 岗位性质 | 社招/校招、正式/外包 |

### Step 2: 确定搜索半径 + 地图扫描

**关键**：当用户提到具体地址时：

1. 先用 WebSearch 搜 `{地址} 附近 科技公司 园区 产业园` 确定周边产业集聚区
2. 对周边科技园区的入驻企业进行名录搜索：`{园区名} 入驻企业 科技公司`
3. 识别附近所有潜在公司，建立「周边企业清单」
4. 对周边企业，用 Google Maps / 百度地图类搜索确定大致距离（可估算"步行/骑行/驾车X分钟"）

**距离估算方法**（粗粒度）：
- 已知公司地址在目标地址同街道/同园区 → 标注 "步行可达" / "~500m"
- 同区不同街道 → 搜索 `{公司名} 地址 滨江区` 获取门牌号，估算距离
- 无法精确定位 → 标注 "滨江区内" 作为最粗粒度

### Step 3: 多渠道并行搜索（官网优先 + 平台交叉验证）

**核心策略：先扫官网发现平台没有的独家岗位，再交叉验证去重。**

#### Phase A: 官网独有岗位扫描（优先级最高）

对 Step 2 中识别的周边企业逐家执行：

1. 加载 `references/company_career_pages.md`，查目标公司官方招聘入口
2. 对每家公司，WebSearch 搜索：
   - `{公司名} 官网 社会招聘 {岗位}` — 确认是否有岗位
   - `{公司名} 社招 {岗位} 官网 site:{公司域名}` — 精准搜官网招聘页
3. 对确认有招聘入口的，用 WebFetch 抓取官网招聘页面提取岗位信息
4. **重点标注**：凡是只在官网出现、未在猎聘/BOSS/智联上看到的岗位，标记为 `⭐官网独家`

#### Phase B: 平台交叉验证（去重用）

在完成官网扫描后，对主流平台进行搜索：

- `{公司名} {岗位} {城市} BOSS直聘`
- `{公司名} {岗位} {城市} 猎聘`
- `{公司名} {岗位} {城市} 智联招聘`
- 至少覆盖 BOSS直聘 + 猎聘 + 智联招聘 三个平台

**去重逻辑**：
- 同一公司同一岗位，官网有 → 优先用官网链接，来源标「官网」
- 同一公司同一岗位，官网无 → 取平台的详情页链接，来源标对应平台名
- 某公司只在官网有岗位、平台搜不到 → 标记「官网独家」+ 高亮

#### Phase C: 提取直达链接

- 官网：提取招聘系统中具体岗位的落地页 URL
- 平台：取 `zhipin.com/job_detail/xxx`、`liepin.com/job/xxx`、`zhaopin.com/jobdetail/xxx` 等详情页
- 每条岗位记录必须关联一个可点击直达的链接

### Step 4: 时效过滤

- 搜索时关注搜索结果中是否包含发布日期/更新时间
- **只纳入近 3 个月（2026年3月12日至今）发布或更新的岗位**
- 对于搜索结果显示"发布于X天前"、"X周前更新"的，换算确认是否在3个月内
- 无法确定发布时间的岗位，标注 `更新时间: 未知`
- 明显过期（如显示"半年前"、"2025年"）的岗位直接排除

### Step 5: 整理输出

**排序规则**（多级排序）：

1. **距离优先**：离用户地址最近的公司排前面
2. **官网独家优先**：只在官网招聘的岗位优先于平台岗位
3. **同一公司按岗位层级降序**：高级 > 资深 > 普通

**输出格式**（强制）：

```markdown
## {地点} {岗位} 社招岗位汇总

> 📍 基准地址：{用户地址} | 🕐 数据采集：{当前日期} | ⏳ 仅展示近3个月在招岗位

| 序号 | 公司 | 距离 | 岗位 | 薪资 | 经验 | 更新时间 | 来源 | 投递链接 |
|:---:|------|------|------|------|------|----------|------|----------|
| 1 | XX公司 | ~500m | Java开发 | 15-25k | 3-5年 | 2026-06 | 官网 | [点击投递](url) |
|   |      |       | 高级Java | 25-35k | 5年+ | 2026-05 | 官网 | [点击投递](url) |
| 2 | YY公司 | ~1.2km | Java后端 | 20-30k | 3-5年 | 2026-05 | BOSS直聘 | [点击投递](url) |
```

**单元格合并规则**（强制执行）：
- 同一家公司有多个岗位时，第一行之后的行的「序号」和「公司」和「距离」列留空
- 不同公司之间用空行或用分割线区分视觉层次
- 合并仅限「序号」「公司」「距离」三列，其余列每行独立

**关键列说明**：

| 列名 | 要求 |
|------|------|
| 距离 | 从用户地址到公司的估算距离（如 ~500m / ~1.2km / ~3km / 滨江区内） |
| 来源 | 平台名称：官网、BOSS直聘、猎聘、智联招聘、51job。官网独家的标注 `⭐官网` |
| 更新时间 | 岗位发布/更新日期，格式 YYYY-MM；未知填 `—`；超过3个月的不展示 |
| 投递链接 | 具体岗位详情页 URL，可点击直达 |

## 关键原则

1. **地图优先**：先扫周边公司官网，发现平台没有的独家岗位
2. **链接直达岗位**：所有输出链接必须指向具体岗位详情页
3. **官网优先 + 标注来源**：每条岗位标注来源渠道，官网独家特别标记
4. **时效过滤**：仅展示近 3 个月发布/更新的岗位
5. **距离可视化**：每条岗位附带距离信息
6. **单元格合并**：同公司多岗位合并「序号」「公司」「距离」列
7. **不编造**：只有 WebSearch/WebFetch 确认存在的岗位才写入
8. **外包标注**：外包/OD岗位需在来源列或备注中明确标注「外包」
